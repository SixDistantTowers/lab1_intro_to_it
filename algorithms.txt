1. Подготовительный этап (Описание структуры)

    Создать структуру под названием car, которая будет описывать автомобиль. В неё входят три поля: марка (строка), цена (целое число) и год выпуска (целое число).

    Добавить метод (внутреннюю функцию) для этой структуры, который умеет красиво выводить данные об автомобиле на экран.

2. Ввод данных

    Объявить массив из 5 автомобилей.

    Запустить цикл (повторить 5 раз):

        Попросить пользователя ввести марку, цену и год для каждого автомобиля.

        Сохранить эти данные в соответствующие ячейки массива.

    Запросить критерий поиска: спросить у пользователя минимальный год выпуска (myear). Программа будет искать машины, выпущенные в этот год или позже.

3. Инициализация поиска

    Установить начальные значения:

        Переменную для хранения минимальной цены (minprice) сделать очень большой (например, 9 999 999), чтобы любая реальная цена была меньше неё.

        Создать переменную-индекс для хранения позиции найденного автомобиля.

        Создать логический флаг (startedonce), изначально равный «ложь». Он покажет, нашли ли мы вообще хоть одну машину, подходящую под условие года.

4. Алгоритм поиска (Цикл)

    Просмотреть каждый автомобиль в массиве по очереди:

        Проверить условие по году: Если год выпуска текущего автомобиля больше или равен введенному пользователем (myear), то переходим к следующему шагу.

        Активировать флаг: Если условие по году выполнено, отметить, что подходящая машина найдена (startedonce = true).

        Сравнить цену: Если цена текущего автомобиля меньше текущего значения minprice:

            Обновить minprice, записав туда цену этой машины.

            Запомнить индекс (позицию) этой машины в массиве.

5. Вывод результата

    Проверить флаг:

        Если хотя бы одна машина подошла по году (startedonce истинно) — вызвать метод вывода данных для машины, чей индекс мы запомнили.

        Если ни одна машина не подошла по году — вывести сообщение «No cars found».


1. Подготовительный этап (Описание структуры)

    Определить глобальную переменную curmonth, которая хранит текущий месяц (в коде это 5, т.е. май).

    Создать класс film (Фильм), в котором:

        Закрытые поля (private): название, киностудия, режиссер (продюсер), день, месяц и год выхода.

        Конструктор: специальная функция, которая сразу при создании объекта заполняет все его поля данными.

        Дружественная функция (friend): объявить внешнюю функцию getinfo, давая ей разрешение «заглядывать» в скрытые (private) данные класса.

2. Описание логики фильтрации (Функция getinfo)

Алгоритм работы функции проверки выглядит так:

    Принять данные: функция получает конкретный объект фильма и строку с именем режиссера, которую ввел пользователь.

    Проверить условия:

        Совпадает ли имя режиссера в объекте фильма с тем, что ввел пользователь?

        Совпадает ли месяц выхода фильма с текущим месяцем (curmonth)?

    Вывести результат: если оба условия выполнены (используется логическое «И»), вывести на экран все данные об этом фильме (название, студию, режиссера и полную дату). Если условия не совпали — ничего не выводить.

3. Основной процесс выполнения (Функция main)

    Инициализация: Создать несколько объектов (экземпляров) класса film с заранее прописанными данными о мировых шедеврах кино.

    Ввод пользователя:

        Вывести на экран запрос «Enter the producer's name».

        Считать введенную строку целиком (с учетом пробелов) и сохранить её в переменную producerName.

    Обработка: Поочередно вызвать функцию getinfo для каждого созданного фильма, передавая в неё сам фильм и имя режиссера для сравнения.
